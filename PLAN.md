# PPIO 产业政策信息流 — 项目实施计划（45天）

## 项目定位

PPIO 产业政策信息流是 PPIO GR 部门的内部情报工具，服务于三个核心场景：

1. **政策追踪**：每日自动扫描国家及上海市级政府政策动向，为 GR 接待、申报材料、V-team 汇报提供一手素材
2. **竞品监控**：持续追踪无问芯穹、硅基流动、七牛云、优刻得及海外 Baseten/Fireworks AI 等竞品的融资、产品、战略变化
3. **IPO 合规感知**：跟踪 VIE 架构审核、港交所上市监管、境外备案新规等与公司上市进程直接相关的监管动态

## 当前状态（Day 0，已上线）

### 技术架构

```
01-fetch.mjs      → 多源抓取（RSS + Google News + 上海政府页面）
02-curate.mjs     → DeepSeek V4 Pro AI 分类（信号标签 + 车道路由）
03-synthesize.mjs → AI 生成日报主线 + 风向指标
04-render.mjs     → 静态 HTML 渲染
```

- **运行方式**：macOS launchd 每日 08:00 自动触发，开机后补跑
- **发布方式**：自动 git push → GitHub Pages 静态托管
- **访问地址**：https://alexzeng-eng.github.io/ppio-expresso/

## 第一阶段：内容打磨（Day 1–15）

**目标：让每日内容质量达到可直接用于 GR 汇报的水准**

### 内容质量
- 每日精选条数从 7-10 条提升至 12-15 条
- attend 条目摘要增加"对 PPIO 的具体影响"段落（目标 250-300 字）
- 竞品类条目自动补充竞品背景（融资轮次、产品定位、与 PPIO 重叠度）

### 信源扩展
- 补充上海市数据局、浦东新区科经委、张江科学城政府页面抓取
- 加入半导体行业观察、甲子光年的 Google News 搜索覆盖
- 优化 DeepSeek 分类 prompt，减少 category 误判

### 页面优化
- 移动端响应式适配（手机查阅）
- 存档页增加分类筛选

## 第二阶段：自动化提升（Day 16–30）

**目标：减少人工干预，关键事件主动推送**

### 企业微信推送通知
当出现以下事件时，自动推送到 GR 群：
- 国常会/政治局级别政策出台（🏛️ 信号）
- 直接竞品融资超过 1 亿元（⚔️ 信号）
- VIE/港交所监管新规发布（📜 信号）
- 英伟达芯片出口管制重大变化（🌏 信号）

实现：pipeline 末尾检测高优先级 attend 条目，触发企业微信 Webhook。

### 竞品档案页
为主要竞品建立独立档案（无问芯穹、硅基流动、超聚变、优刻得）：
- 融资历史时间线
- 产品定位演变
- 与 PPIO 重叠度分析
- 每次出现相关新闻时自动更新

### 历史趋势
存档页增加：
- 各分类信号频次周趋势折线图
- 风向指标历史对比（政策热度走势）

## 第三阶段：GR 场景深化（Day 31–45）

**目标：从信息工具升级为 GR 工作辅助系统**

### 周报自动生成
每周一自动生成可直接使用的周报，包含：
- 本周主线（1 段，≤150 字）
- 重点政策解读（2-3 条，含 PPIO 映射）
- 竞品动态汇总表
- 风向指标趋势（与上周对比）
- 下周关注事项

输出格式：HTML 网页版 + 可复制的纯文本版（方便粘贴到微信/飞书）

### GR 接待辅助
接待前根据来访单位自动生成简报：
- 该部门近 30 天政策动向摘要
- PPIO 与该部门政策的契合点
- 建议沟通要点 3 条

### 英文摘要
海外监管类内容自动生成英文摘要，方便与投行、律所沟通（IPO 路演准备）。

## 技术风险与应对

| 风险 | 当前状态 | 应对措施 |
|------|---------|---------|
| Google News 限流 | 偶发 | 并发控制（3并发），可降至 1 |
| DeepSeek API 不稳定 | 有 rule-based fallback | 备选 Claude API |
| GitHub Pages 国内访问慢 | 偶发 | 可迁移至 Cloudflare Pages |
| 单机依赖（Mac 休眠） | launchd 开机补跑 | 可迁移至云服务器 |
| 信源质量参差 | 评分过滤 | 持续调整关键词权重 |
